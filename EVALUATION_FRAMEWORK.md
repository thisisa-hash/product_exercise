# Product Manager Assessment — Evaluation Framework

## Context

This framework evaluates candidate take-home assignments for the **Product Manager (THA)** role at Novibet. The assignment has **4 exercises**, each with specific deliverables. Two calibration benchmarks inform the scoring: one "good" candidate and one "bad" candidate, plus the hiring manager's feedback on each.

---

## The Assignment (Summary)

### Exercise 1: Google Analytics — Offers Page
- Define events, KPIs, funnels for the Offers Page
- Provide illustrative reports (random numbers acceptable)
- Define what success looks like
- Propose additional KPIs to enhance decision-making
- **Source file:** `Offers Page for THA PM_MX.xlsx` (Screenshots Desktop + Apps)

### Exercise 2: Data Analysis — Sportsbook Missions
- Create analytical reports from 13-week Mexico data
- Identify performance trends and pain points
- Propose additional KPIs
- Define what success looks like
- Suggest product improvements based on data
- Consult the product team on next steps; respond to "drop the product" scenario
- **Source file:** `Sportsbook Missions for THA PM_MX.xlsx` (Definitions, Conversion, Customer Base, Category)

### Exercise 3: How To? — Casino Loyalty Program
- Frame an analysis to assess if the loyalty program is successful (Jan–Aug 2025)
- Define perimeter: time period, customer segment, etc.
- Provide visuals if necessary
- **Key product detail:** No opt-in required. Players automatically receive coins. Premium is one-off; VIP is monthly recurring.

### Exercise 4: North Star Metric Framework
- Propose an NSM for one of the 3 products
- Explain rationale
- Identify 3–4 secondary metrics
- Describe how the NSM guides product decisions and prioritization

### Deliverables Expected
- Report (PDF or PowerPoint) with analysis, visualizations, and recommendations for all 4 tasks
- Additional files used in analysis (e.g., Excel spreadsheets)
- Brief (max 500 words): reflection on approach, assumptions, areas for further exploration

### Official Evaluation Criteria (from the assignment)
1. Simplicity (above all)
2. Depth and accuracy of data analysis
3. Quality and clarity of visualizations
4. Understanding the application of the North Star Metric framework
5. Ability to connect data insights to business goals
6. Creativity and feasibility of proposed solutions

## Evaluation Rubric

### Tier 1: Product Thinking & Language (Most Important — 35%)
| Score | Description |
|-------|-------------|
| 5 | Speaks the PM language naturally. Connects features to user value and business goals. Thinks in terms of product lifecycle. |
| 4 | Demonstrates solid product thinking with minor gaps. |
| 3 | Shows some product awareness but defaults to analytical/generic framing. |
| 2 | Surface-level product understanding. Misses key product mechanics. |
| 1 | No evidence of PM thinking. Treats exercises as pure data tasks. |

**Red flags:** Misunderstanding product mechanics (e.g., loyalty has no opt-in), analyzing the wrong product (e.g., slot game instead of Offers Page), irrelevant KPIs (e.g., RTP for Offers Page).

### Tier 2: Definition of Success & Strategic Framing (25%)
| Score | Description |
|-------|-------------|
| 5 | Clear, multi-dimensional success definition (awareness, engagement, conversion, financial). Proposes concrete thresholds. |
| 4 | Good success definition with most dimensions covered. |
| 3 | Generic success definition, lacks specificity or measurability. |
| 2 | Vague or incomplete. |
| 1 | Missing or incoherent. |

**Good signals:** Tiered success (awareness → engagement → conversion → revenue), specific metric targets, connecting success to both user value and business value.

### Tier 3: Depth & Accuracy of Analysis (20%)
| Score | Description |
|-------|-------------|
| 5 | Deep engagement with the data. Correct calculations. Identifies non-obvious patterns and pain points. |
| 4 | Solid analysis with minor errors. Key trends identified. |
| 3 | Reasonable analysis but shallow. Obvious observations only. |
| 2 | Superficial or contains significant errors that undermine insights. |
| 1 | Did not meaningfully engage with the provided data. |

**Note from calibration:** Calculation errors alone do not disqualify a good candidate IF the product thinking is strong. However, using wrong data or analyzing the wrong thing is a bigger problem.

### Tier 4: Presentation Quality & Clarity (10%)
| Score | Description |
|-------|-------------|
| 5 | Fun to read. Clear structure. Charts have proper labels, titles, axes. Professional quality. |
| 4 | Well-organized, minor presentation issues. |
| 3 | Adequate but not engaging. Some unclear charts or formatting. |
| 2 | Noisy charts, missing labels, hard to follow. |
| 1 | Poorly organized, confusing, or incomplete. |

### Tier 5: NSM Framework Understanding (10%)
| Score | Description |
|-------|-------------|
| 5 | Excellent NSM choice with clear rationale connecting user value, product growth, and business value. Secondary metrics well-chosen. Practical guidance on using the framework. |
| 4 | Good NSM choice and rationale with minor gaps. |
| 3 | Reasonable NSM but rationale is thin or secondary metrics are weak. |
| 2 | NSM chosen but framework understanding is superficial. |
| 1 | Missing or fundamentally misunderstands the NSM concept. |

**Good signals:** Choosing a metric that reflects BOTH user value and business goals (e.g., rewards redeemed reflects player value received AND correlates with play/bet volume). Explaining how it guides prioritization.

**Calibration note:** The good candidate chose Casino Loyalty (strategic, showed understanding of both sides). The bad candidate chose Sportsbook Missions CR (more operational, less strategic depth).

## Exercise-Specific Checkpoints

### Exercise 1: Offers Page — What to Look For
- [ ] Events categorized meaningfully (conversion, engagement, navigation)
- [ ] KPIs are relevant to an Offers Page (impressions, CTR, bounce rate, redemption rate, time on page)
- [ ] Funnels are detailed with steps (not just named abstractly)
- [ ] Illustrative reports provided with numbers (even fake ones)
- [ ] Reports structured by funnel stage (top/mid/low) or similar logical flow
- [ ] Success defined across multiple dimensions (not just "high conversion")
- [ ] Additional KPIs include business metrics (ROI, CAC, ARPU, CLV)
- [ ] Demonstrates understanding of the Offers Page as a centralized hub for promotions

### Exercise 2: Missions — What to Look For
- [ ] Uses all 4 data tabs (Definitions, Conversion, Customer Base, Category)
- [ ] Identifies the public vs. private mission distinction and its impact on CR
- [ ] Spots the opt-in vs. completion divergence (opt-ins growing, completions declining)
- [ ] Identifies sport category patterns (e.g., basketball correlating with higher CR)
- [ ] Calculations are correct (or at least directionally sound)
- [ ] Charts are clear with proper labels and axes
- [ ] Proposes concrete product improvements grounded in data
- [ ] Success defined in terms of both user engagement and business impact
- [ ] "Drop the product" response is nuanced (not a flat yes/no)
- [ ] Additional KPIs go beyond what's in the data (bet frequency, ARPU, CLV, retention)

### Exercise 3: Loyalty — What to Look For
- [ ] **CRITICAL:** Understands there is NO opt-in — this is automatic
- [ ] Correctly distinguishes Premium (one-off) vs. VIP (monthly recurring) stages
- [ ] Frames a concrete analysis perimeter (time period, segments, channels)
- [ ] Proposes relevant metrics (milestone progression, redemption rate, churn by stage, CLV)
- [ ] Considers appropriate time horizon for assessment (the program is 8 months old)
- [ ] Mentions comparison cohorts (loyalty vs. non-loyalty, pre vs. post-launch)
- [ ] Provides or describes useful visuals (progression curves, retention cohorts)

### Exercise 4: NSM — What to Look For
- [ ] NSM reflects user value AND business growth (not just an operational metric)
- [ ] Clear rationale explaining why this metric was chosen over alternatives
- [ ] 3–4 secondary metrics that genuinely complement the NSM
- [ ] Practical description of how the NSM guides decisions and prioritization
- [ ] Shows strategic thinking about the chosen product

## How to Use This Framework

1. Read the candidate's full submission (all files)
2. Score each of the 5 tiers independently (1–5)
3. Check exercise-specific checkpoints
4. Calculate weighted total: (T1 * 0.35) + (T2 * 0.25) + (T3 * 0.20) + (T4 * 0.10) + (T5 * 0.10)
5. Map to overall rating
6. Write a narrative summary highlighting:
   - Top 3 strengths
   - Top 3 weaknesses
   - Comparison to calibration benchmarks
   - Final recommendation with confidence level
